# 裝置監控精靈（Device Monitor）

> **繁體中文** | [English](README.md) | [架構與開發說明](ARCHITECTURE.md)

即時監控多台 Android 手機的電量與網路狀態，電量低於閾值時自動通知主裝置。  
無需自架伺服器，使用 Google 帳號登入即可直接使用。

---

## 功能

### 監控與顯示

- 即時顯示所有裝置的電量、充電狀態、網路類型（Wi-Fi / 4G / LTE / 5G NSA / 5G SA）
- Wi-Fi 顯示 SSID，行動網路顯示電信商名稱與訊號強度（格數 + dBm）
- **電量歷史圖**：每張裝置卡片顯示最近 5 筆電量趨勢圖（可在清單設定中開關）
- **首頁 Widget**：使用 Jetpack Glance 製作的桌面小工具，直接在桌面查看裝置狀態
- **Realtime 連線狀態**：WebSocket 斷線時顯示提示橫幅，重連後自動消失
- **排序方式**：點擊排序按鈕切換排列方式——依名稱（預設）、電量升序、電量降序、離線優先（stale 或離線裝置浮至頂端）
- **清單顯示設定**（標題列 Tune 按鈕）：個別開關警告閾值滑桿、緊急閾值滑桿、電量歷史圖的顯示；也是邀請夥伴的入口

### 警報與通知

- 低電量警報：電量低於警告閾值時，主裝置收到本地通知（充電中自動靜音，拔除充電後若仍低於閾值立即補發）
- **雙層警報閾值**：每台裝置各自設定警告閾值（10% 間隔，10–100%）與緊急閾值（預設為警告閾值的一半，最低 10%），以不同顏色區分
- **充滿電通知**：裝置充到 100% 時發送通知
- **裝置離線通知**：裝置超過 3 分鐘未回報時發送通知
- **FCM 推播通知**：透過 Firebase Cloud Messaging，即使 App 在背景或已關閉也能收到警報
- 主裝置設定：指定一台接收所有警報的主裝置（同帳號唯一）
- **免打擾時段**：設定特定時間段，此段時間內所有通知自動靜音
- 裝置別名：為每台裝置設定自訂顯示名稱（通知也使用別名）

### 監控清單

- **置頂裝置**：本機永遠排第一；可右滑其他裝置卡片置頂，置頂裝置支援長按拖曳排序
- **刪除裝置**：在設定中開啟刪除模式，每張卡片右滑後會出現刪除按鈕；支援多選——點擊卡片勾選，再點擊 Header 按鈕批次刪除；最後一次刪除的 60 秒後自動關閉刪除模式
- **批次設定警報閾值**：在刪除模式下選取多台裝置，可一次設定所有被選裝置的警告閾值
- 下拉重新整理：向下滑動即可強制重新載入所有裝置資料

### 夥伴模式

- 跨帳號共享裝置監控——產生 8 碼邀請碼（附 QR Code），邀請碼有效期 30 分鐘（倒計時顯示）
- 可對每位夥伴個別管理哪些裝置要分享；支援自訂夥伴顯示名稱與裝置別名（別名也用於低電量通知）
- **從監控清單直接分享**：右滑裝置卡片即可對每位夥伴個別切換分享狀態，無需切換頁籤
- 每位夥伴可獨立設定是否接收對方各裝置的低電量警報
- **夥伴分享通知**：夥伴分享新裝置給你時發送通知
- **離線快取**：WebSocket 短暫斷線時，夥伴裝置仍顯示最後已知狀態
- 每個帳號最多 5 位夥伴

### 穩定性

- 背景持續監控，即使螢幕關閉也能持續上傳狀態
- **指數退避重試**：upsert 失敗時自動以遞增間隔重試，應對短暫網路錯誤
- **APP 內更新**：啟動時自動檢查新版本，可直接在 APP 內下載並安裝 APK，無需離開應用程式
- **Beta 更新頻道**：可在設定中開啟，追蹤帶有 `-beta` 字樣的測試版

## 安裝需求

- Android 10（API 29）以上
- Google 帳號
- 允許「安裝未知來源應用程式」（側載 APK）

---

## 安裝步驟

1. 下載最新 `app-release.apk`
2. 在手機上開啟 APK 並安裝
3. 啟動 App → 點擊「透過 Google 登入」
4. 登入後點擊「啟動監控服務」

每台要監控的手機都重複以上步驟，使用**同一個 Google 帳號**登入。

---

## 設定主裝置

接收低電量警報的手機需要設定為主裝置：

1. 開啟 App → 「我的裝置」頁籤
2. 開啟「主裝置」開關

同一帳號下只能有一台主裝置。

---

## 監控清單操作

| 操作 | 說明 |
|---|---|
| 右滑裝置卡片 | 顯示大頭針、分享按鈕（開啟刪除模式時同時顯示刪除按鈕） |
| 長按置頂裝置的拖曳把手 | 垂直拖動調整置頂順序 |
| 點擊卡片 | 展開 / 收合詳細資訊（刪除模式下改為切換選取狀態） |
| 點擊鉛筆圖示 | 設定裝置別名 |
| 點擊 Header 的「刪除（N）」按鈕 | 批次刪除所有選取的裝置（刪除模式開啟且至少選取一台時顯示） |
| 點擊 Header 的排序圖示 | 切換排序方式：依名稱 / 電量升序 / 電量降序 / 離線優先 |
| 點擊 Header 的 Tune 圖示 | 開啟清單顯示設定：開關閾值滑桿與電量歷史圖；邀請夥伴入口 |
| 下拉清單 | 強制重新整理所有裝置資料 |

本機裝置永遠固定在清單最頂端，無法被其他裝置超過。

## 夥伴模式

夥伴模式允許兩個使用**不同 Google 帳號**的使用者互相監控對方的裝置。

### 新增夥伴

1. 開啟**夥伴**頁籤 → 點擊「邀請夥伴」
2. 選擇要分享的裝置（可一鍵全選），點擊「產生邀請碼」
3. 將 8 碼邀請碼或 QR Code 分享給對方
4. 對方開啟**夥伴**頁籤 → 「輸入邀請碼」→ 手動輸入，或點擊 QR 圖示掃描
5. 配對完成，雙方可在各自的清單中看到對方共享的裝置

### 管理分享裝置

- **從監控清單分享**：在監控清單右滑任何裝置卡片，即可看到分享按鈕，對每位夥伴個別 toggle 是否分享，無需切換頁籤
- **事後追加裝置**：在夥伴頁籤的夥伴卡片上點擊「管理分享」，選取更多裝置（支援一鍵全選）
- **取消分享**：點擊夥伴卡片「我分享給他的裝置」列表中的 × 按鈕

### 命名與通知

- **重新命名夥伴**：點擊夥伴卡片標題旁的鉛筆圖示，設定自訂顯示名稱
- **設定裝置別名**：點擊共享裝置旁的鉛筆圖示，設定自訂名稱；低電量通知也會使用此別名
- 每位夥伴可獨立設定是否接收對方各裝置的低電量警報

每個帳號最多 5 位夥伴。

## 隱私說明

- 所有裝置資料儲存於 Supabase，並透過 Google 帳號 UID 隔離
- 不同帳號的資料彼此完全獨立，無法互相存取
- 不收集任何個人身份識別資訊以外的資料

## 維護者備忘

### 強制重新登入 Flag

**檔案：** `app/src/main/java/tw/bluehomewu/devicemonitor/AppConfig.kt`

```kotlin
object AppConfig {
    const val FORCE_RESIGN_FROM_VERSION: String? = "1.13.0"
}
```

將 `FORCE_RESIGN_FROM_VERSION` 設為版本字串後，所有已登入的舊使用者在下次啟動 App 時會被強制登出並重新登入，同時顯示 Toast 提示。

**使用時機：** 後端遷移（例如切換至自架 Supabase）、Auth schema 異動，或其他會讓既有 session 失效的情況。

**如何在下一版觸發強制重登：**

1. 開啟 `AppConfig.kt`
2. 將 `FORCE_RESIGN_FROM_VERSION` 改為新版本號（例如 `"1.14.0"`）
3. 發布新版本

**行為說明：**
- 有既有 session 的使用者：首次啟動時自動登出，顯示 Toast：*「版本更新後需重新登入，請重新登入」*
- 全新安裝的使用者：**不受影響**（沒有 session 可清除）
- 使用者重新登入後，版本號會記錄在 SharedPreferences（key：`force_resign_done_for`），後續啟動不再觸發
- 設為 `null` 可完全停用此功能

**未來再次觸發：** 只需將值改為當時的版本號（例如 `"1.15.0"`），所有存入值與新版本號不符的使用者都會再次被強制重登一次。
